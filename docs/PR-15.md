# PR 15 - Caesar II Candidate DAT/CFG/IX Recon Probe Layer

PR 15 adds a bounded reconnaissance probe path for high-value Caesar II non-map candidates before any semantic parsing work.

## Scope

- ✅ adds a generic candidate probe/report utility in the data layer
- ✅ supports bounded loading of arbitrary selected candidate files
- ✅ reports stable structural hints: size, leading signature, first LE scalars, coarse kind classification
- ✅ adds bounded text preview for obvious text/config candidates (such as `RESOURCE.CFG`)
- ✅ adds paired DAT/IX reconnaissance reporting (for examples like `DISCS.DAT` + `DISCS.IX`)
- ❌ does not assign semantic meaning to contents
- ❌ does not implement real parsers for `C2MODEL.DAT`, `REGIONS.DAT`, `DISCS.*`, `HISTORY.DAT`
- ❌ does not add rendering/simulation logic

## Changes

### 1) New candidate probe utility (`romulus::data`)

Added `candidate_probe.{h,cpp}` with:

- bounded load + probe entrypoint:
  - `probe_candidate_files(data_root, candidates, max_file_load_bytes)`
- deterministic, reconnaissance-focused report model:
  - per-file path, byte size, signature (hex + printable), first `u16/u32` LE scalars
  - coarse file kind classification:
    - `text-like`
    - `mz-executable-like`
    - `opaque-binary`
- optional bounded text preview metadata/content for text-like or obvious config files
- optional DAT/IX pair summary block when both sides are selected

The implementation deliberately reuses existing primitives:

- `load_file_to_memory` for safe bounded file reads
- `probe_loaded_binary` for initial scalar/header probing

### 2) CLI support for selected candidate probing

`apps/caesar2` now accepts repeatable `--probe-candidate <path>` arguments.

Examples:

- `--probe-candidate RESOURCE.CFG`
- `--probe-candidate C2MODEL.DAT`
- `--probe-candidate REGIONS.DAT`
- `--probe-candidate DISCS.DAT --probe-candidate DISCS.IX`
- `--probe-candidate HISTORY.DAT`

Mutual exclusion with legacy `--probe-file` is enforced for clarity.

### 3) Tests

Added `romulus_candidate_probe_tests` coverage for:

- text-like config probing + bounded preview behavior
- MZ-like detection path
- DAT/IX pair reporting path
- bounded max-size load rejection

## Definition of done mapping

- selected candidate files can be safely loaded and reported: ✅
- text-like files get bounded preview: ✅
- binary candidates get stable structural probe output: ✅
- DAT/IX pair reporting works at reconnaissance level: ✅
- no speculative semantics are introduced: ✅
