# PR 10 - Minimal SDL Debug Viewer for a Single Decoded Caesar II Image

## Summary
PR 10 adds a narrow debug viewer path that displays one decoded Caesar II indexed image using a parsed palette in an SDL window.

This implementation is intentionally scoped to one image at a time and avoids introducing rendering architecture, atlases, animation systems, or broader runtime abstractions.

## What was added

### 1) Minimal viewer path
- Added an optional debug-view image in platform application options.
- Added a small SDL renderer/texture path used only when a debug-view image is provided.
- Uploads decoded RGBA pixels to a single `SDL_TEXTUREACCESS_STATIC` texture.
- Draws the texture each frame with simple clear/copy/present logic.

### 2) CLI support for viewing one decoded image
- Added:
  - `--view-tile-file <path>`
  - `--view-palette-file <path>`
  - existing `--index-zero-transparent` support also applies to viewer mode
- Viewer mode is mutually exclusive with export mode.
- Invalid viewer argument combinations fail early with clear errors.

### 3) Helper utilities
- Added a shared decode helper in `apps/caesar2/main.cpp` that reuses existing:
  - bounded file loading
  - palette parsing
  - indexed image decoding
  - palette-to-RGBA conversion
- Added a small viewer window layout utility to compute integer scale/window size for debug display.

### 4) Tests
- Added tests for viewer window layout helper:
  - scales up small images
  - clamps scale to 1 for large images
  - returns default layout on invalid inputs

## Usage

```bash
./caesar2 \
  --data-dir /path/to/caesar2 \
  --view-tile-file CITSINGLE/TILE_0001.BIN \
  --view-palette-file CITSINGLE/PALETTE.COL
```

Optional transparency behavior for index 0:

```bash
./caesar2 \
  --data-dir /path/to/caesar2 \
  --view-tile-file CITSINGLE/TILE_0001.BIN \
  --view-palette-file CITSINGLE/PALETTE.COL \
  --index-zero-transparent
```

## Error handling
- Malformed or unreadable tile/palette inputs fail cleanly through existing parse/load result paths.
- Viewer mode argument validation rejects partial/mixed mode invocation.

## Scope notes
- Single decoded image only.
- No atlas, animation, batching, scene graph, or generalized renderer system.
- Intended for debug inspection and iterative reverse-engineering workflows.
