# PR 33 — Win95 PACK ILBM success index + bounded export

## Summary
This PR extends the Win95 `PACK` ILBM tooling with a deterministic **successful-parse index** and narrow export paths that only operate on ILBM entries already proven valid by the existing bounded parser flow.

Scope is intentionally limited:
- no new payload families
- no general archive extraction framework
- no gameplay semantic labeling

## Added data model support
- `Win95PackIlbmIndexEntry`
  - `entry_index`, `offset`, `size`
  - parsed image dimensions (`width`, `height`)
  - optional `palette_color_count`
  - compact `classification_hint`
- `Win95PackIlbmIndex`
  - total/candidate/successful counters
  - deterministic `successful_entries` list
- `Win95PackIlbmExportResult`
  - per-entry export status for bounded batch export reporting

## Index construction
A new helper builds a stable success index from PR 32 batch analysis results:
- `build_win95_pack_ilbm_success_index(...)`

Only entries with `parse_success == true` are included.
Ordering is deterministic by PACK entry index because PR 32 result ordering is deterministic.

## Index/report helpers
- `find_win95_pack_ilbm_index_entry(...)` allows explicit success-index membership checks.
- `format_win95_pack_ilbm_index_report(...)` provides stable reporting with:
  - total entries
  - possible ILBM candidate count
  - successful ILBM count
  - bounded preview (default)
  - optional all-entry mode

## CLI additions
- `--index-pack-ilbm <container>`
- `--index-pack-ilbm-entries-all`
- `--export-pack-ilbm-success <container> <entry_index> --export-output <path>`
- `--export-pack-ilbm-first <container> <count> --export-output-dir <dir>`

### CLI behavior
- New modes remain mutually exclusive with unrelated probe/view/export modes.
- `--export-pack-ilbm-success` only exports entries present in the success index.
- `--export-pack-ilbm-first` is bounded to at most 64 exports and prints per-entry deterministic result lines.
- Export uses the existing ILBM -> RGBA -> PPM path.

## Error behavior
Explicit failures are emitted for:
- invalid entry index
- entry not present in successful ILBM index
- malformed/missing export output path or output directory
- bounded batch export count violations

## Tests
`tests/data/win95_data_container_tests.cpp` now covers:
- synthetic PACK with multiple successful ILBM entries
- success-index construction and deterministic order
- stable index report formatting and bounded preview behavior
- valid single-entry export path via shared PPM exporter
- invalid index lookup failure
- non-success-entry export/index lookup failure
