# PR 27 - Startup setup wizard for Caesar II Win95 data root

## Summary

PR 27 replaces the one-shot bootstrap prompt from PR 26 with a minimal first-run setup wizard state flow that stays data-independent and startup-focused.

Implemented wizard states:

1. `WizardWelcome`
2. `WizardChooseFolder`
3. `WizardValidating`
4. `WizardValidationFailed`
5. `WizardConfirm`
6. `WizardDone`

## Implementation notes

- Added a reusable setup wizard state machine in `romulus::platform::run_setup_wizard`.
- Reused existing Win95 data-root validation via `romulus::data::validate_data_root`.
- Reused existing persistence helpers (`load_persisted_data_root` / `persist_data_root`).
- Kept folder selection on native dialogs from PR 26 (`zenity` / `kdialog`).
- Wizard behavior:
  - missing/invalid root enters wizard
  - invalid selections show required missing entries
  - missing optional Win95 entries are shown as warnings on confirm
  - canceled folder picker returns to choose step (no crash)
  - user can retry without restarting
  - successful selection persists config and continues startup

## App integration

- Application startup now delegates setup UX to `run_setup_wizard` via SDL message boxes.
- Command-line `--data-dir` remains preferred when present.
- If selected root is absent/invalid, startup falls back into wizard flow.

## Tests added

Expanded `tests/platform/startup_tests.cpp` with wizard flow coverage:

- first launch with no configured root enters setup path
- valid selected Win95 root reaches confirm/done
- invalid selected root shows validation failed and supports retry
- canceled picker returns to wizard flow cleanly
- persisted valid root skips wizard path by validating as ready
- invalid persisted root re-enters wizard and recovers
