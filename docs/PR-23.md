# PR 23 - Bounded Win95 PE Resource-Section Reconnaissance

## Summary
This PR adds bounded, BinaryReader-based reconnaissance for Win95 PE resource trees in `CAESAR2.EXE`-style binaries.

Scope remains **reconnaissance-only**:
- no disassembly
- no execution
- no deep resource payload decoding

## What changed
- Added PE resource-section models:
  - `PeResourceDirectoryEntry`
  - `PeResourceLeaf`
  - `PeResourceTree`
  - `PeResourceTypeSummary`
  - `PeResourceSectionReport`
- Extended PE parsing to traverse and validate `.rsrc` directory trees using already-validated PE metadata and section mapping.
- Added bounded parsing/validation for:
  - resource root directory
  - directory tables and entries
  - type/name/language levels
  - resource data entries
- Added support for:
  - numeric resource IDs
  - UTF-16 string-name resource entries
- Added stable deterministic formatter:
  - `format_pe_resource_report(...)`
- Added compact summary output:
  - top-level resource type count
  - per-type entry and leaf counts
  - total leaf count
  - known type labels when available (`ICON`, `MENU`, `DIALOG`, `STRING`, `ACCELERATOR`, `VERSION`, etc.) while preserving raw IDs
- Added CLI mode:
  - `--probe-exe-resources <path>`
- Enforced `--probe-exe-resources` mutual exclusion with other probe/export/view modes.
- Added synthetic fixture tests for:
  - valid minimal resource tree traversal
  - malformed/truncated resource structures
  - numeric ID and string-name entries
  - stable report content

## Guardrails and safety
- Bounded traversal limits on directories and leaves.
- Explicit structured failures on malformed offsets and truncated structures.
- No third-party PE libraries.
- No payload semantic inference beyond size/RVA/offset reconnaissance.
