# PR 37 ŌĆö Unified Win95 PACK known-success index/report (ILBM + text + simple PL8)

This PR adds a deterministic **unified known-success index** for Win95 `PACK` containers across the currently supported extracted families:
- ILBM-like entries
- text-like entries
- simple PL8-like entries (shared 768-byte `.PL8` layout)

The purpose is to provide one compact coverage view for ŌĆ£what we can currently decode successfullyŌĆØ without adding any new payload families.

## What was added

### Data models
Added unified index/report models in `win95_data_container`:
- `Win95PackKnownFamilyKind`
- `Win95PackUnifiedSuccessEntry`
- `Win95PackUnifiedSuccessSummary`
- `Win95PackUnifiedSuccessIndex`
- `Win95PackUnifiedSuccessReportOptions`

Each unified success entry records:
- PACK entry index
- offset
- size
- extracted family kind (`ilbm`, `text`, `pl8`)
- compact family metadata only:
  - ILBM: width/height + palette color count
  - text: line count/character count + bounded preview
  - PL8: palette entry count + supported layout marker in report

### Unified index builder
Added:
- `build_win95_pack_unified_success_index(...)`

Implementation reuses existing family-specific successful paths:
- ILBM via `analyze_win95_pack_ilbm_batch` + `build_win95_pack_ilbm_success_index`
- text via `analyze_win95_pack_text_batch` + `build_win95_pack_text_success_index`
- simple PL8 via existing `extract_win95_pack_pl8_entry` extraction/validation path

No duplicate family decode logic was introduced.

Ordering is deterministic by PACK entry index.

### Unified report formatter
Added:
- `format_win95_pack_unified_success_index_report(...)`

Default report behavior is compact and deterministic, including:
- total PACK entries
- known/success entries
- unknown entries
- known ratio (`known/total`)
- per-family success counts
- bounded preview of successful entries

Optional all-entry mode is supported with `include_all_entries`.

### CLI support
Added CLI modes:
- `--index-pack-known <container>`
- `--index-pack-known-entries-all`

These follow existing PACK command exclusivity and argument validation style.

## Error behavior
- Family extraction failures are not promoted to unified successes.
- Unsupported entries remain unknown.
- No speculative fallback family guessing was added.

## Tests
Extended `win95_data_container_tests` with synthetic mixed-family fixture coverage:
- successful ILBM entry
- successful text entry
- successful simple PL8 entry
- unsupported unknown entry

Added tests for:
- unified summary totals
- per-family counts
- deterministic entry ordering
- stable compact report formatting
- all-entry preview mode behavior
