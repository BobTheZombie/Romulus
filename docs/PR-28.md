# PR 28 - Win95 DATA/DATA0 reconnaissance

## Summary

PR 28 adds a narrow reconnaissance-only probe path for top-level Win95 install entries `DATA` and `DATA0`.

The probe safely reports each target whether it is:

- missing
- a file
- a directory
- another filesystem node type

No container decoding is introduced.

## What is reported

### File targets

For `DATA` / `DATA0` as files, report includes:

- path
- bounded size
- leading signature bytes (`hex` + printable `ascii`)
- first little-endian scalar values (`u16` / `u32` when available)
- coarse kind (`text-like`, `opaque-binary`, `possible-container`, `executable-like`)

### Directory targets

For `DATA` / `DATA0` as directories, report includes:

- path
- bounded scan count
- truncation marker for large directories
- deterministic preview listing (sorted by entry filename)
- per-entry hints where practical (node kind, size/signature/coarse kind for regular files)

### DATA + DATA0 pair summary

A paired summary is emitted with:

- same-kind vs different-kind relationship
- size delta when comparable
- conservative relationship hints

## CLI

Added `--probe-win95-data` mode:

- requires a ready data root (same startup flow as other probe commands)
- mutually exclusive with other command modes
- prints a stable report format for downstream analysis

## Safety and determinism

- all file reads stay bounded via existing loader limits
- no unbounded directory recursion (top-level entries only)
- directory scans are capped and previews are deterministic
- unknown/opaque states are reported explicitly instead of speculative decoding
