# PR-40: Bounded `FORM PBM ` Support for Caesar II Startup Assets

## Summary
This PR extends Romulus' bounded LBM parser to accept both `FORM ILBM` and `FORM PBM ` assets while preserving existing ILBM behavior.

The immediate driver is real Caesar II startup art (`FORUM.LBM`), which is `FORM PBM ` with `BMHD`/`CMAP`/`BODY` chunks.

## Supported FORM subtypes
- `ILBM` (existing path, preserved)
- `PBM ` (new in this PR)

Any other `FORM` subtype fails explicitly and deterministically.

## Supported PBM subset (this PR)
The PBM path is intentionally narrow and deterministic:
- Required chunks:
  - `BMHD`
  - `CMAP`
  - `BODY`
- Duplicate required chunks are rejected.
- BMHD constraints:
  - `width`/`height`: non-zero and bounded (`<= 4096` each)
  - `depth`: `8` only
  - `masking`: `0` only
  - `compression`: `0` (none) or `1` (ByteRun1)
- BODY decode mode:
  - packed 8-bit indexed pixels (`width * height` bytes after optional ByteRun1 decode)

Unsupported layout combinations fail explicitly with parse errors rather than speculative decoding.

## Non-goals
- Supporting every IFF/LBM bitmap dialect or optional chunk variant.
- HAM/EHB or non-8-bit PBM decode variants.
- Masked PBM decode paths.
- Gameplay/map/UI-system expansion (including ImGui).

## Rationale for adding PBM now
Romulus bootstrap rendering now resolves startup assets case-insensitively and can successfully locate real game assets. Real Caesar II startup files have been verified as `FORM PBM `, so bounded PBM decode is required for deterministic, safe startup image rendering using the existing indexed→RGBA pipeline.
