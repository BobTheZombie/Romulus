# PR-17: Narrow Batch Candidate Classification and Comparative Report

## Goal
Add a narrow, deterministic batch classification/report path that reuses the candidate probe and signature registry layers to compare multiple selected files in one pass.

## Scope
- ✅ add a batch classification helper over existing probe + signature matching APIs
- ✅ add CLI support for classifying multiple selected candidate files in one invocation
- ✅ output compact, per-file summaries with:
  - file name
  - top-ranked decoder id
  - confidence bucket
  - short evidence summary
- ✅ optionally include one secondary match when requested and useful
- ✅ add tests for deterministic ordering and report formatting
- ❌ no decoder implementation
- ❌ no gameplay semantics
- ❌ no new raw file parsing beyond existing probe utilities

## API additions
In `romulus::data::signature_registry`:
- `classify_candidate_batch(probe_bundle, include_secondary_matches)`
- `format_batch_classification_report(report)`

The helper:
- evaluates each selected `CandidateFileReport` using existing `match_candidate_signatures`
- emits one stable summary per file
- sorts output by normalized path for deterministic order
- keeps unknown/opaque outcomes explicit instead of speculative guessing

## CLI additions
`apps/caesar2` now supports:
- repeatable `--classify-candidate <path>`
- optional `--classify-include-secondary`

Example:
```bash
caesar2 --data-dir . \
  --classify-candidate RESOURCE.CFG \
  --classify-candidate DISCS.DAT \
  --classify-candidate DISCS.IX \
  --classify-include-secondary
```

## Output shape
```text
# Caesar II Batch Candidate Classification
- file: DISCS.DAT
  top_decoder: Caesar2DatContainer
  confidence: high
  evidence: .dat extension; matching .ix present; +1 more
  secondary: Caesar2IxIndex (medium, score 10)
```

## Definition of done mapping
- multiple candidate files can be classified in one pass: ✅
- output is stable and useful for selecting next decoder targets: ✅
- tests cover formatting and determinism: ✅
