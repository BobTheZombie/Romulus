# PR 26 - Startup Bootstrap for Win95 Caesar II Data Root

## Summary

Romulus no longer hard-fails at startup when no valid Caesar II data root is configured.
Instead, startup now supports a bootstrap setup flow that works before game assets are loaded.

## What changed

- Switched default data-root validation profile to Win95/C2Win95-oriented required entries:
  - `CAESAR2.EXE`
  - `CAESAR2.INI`
  - `DATA/`
  - `SAVE/`
  - `HISTORY.DAT`
- Added optional/expected Win95 entries metadata:
  - `DATA0/`
  - `RAW/`
  - `SMK/`
  - `CAESAR2.INF`
- Added explicit startup-state model:
  - `NoDataRootConfigured`
  - `DataRootInvalid`
  - `DataRootReady`
- Added startup bootstrap flow in the platform app:
  - When startup is missing/invalid, launch a minimal setup window.
  - Show simple instructions with no game-asset dependencies.
  - Press `O` to open folder selection.
  - Press `Esc` or close window to exit cleanly.
- Added native folder-browser integration (Option 1):
  - Uses `zenity` where available.
  - Falls back to `kdialog` where available.
- Added data-root persistence:
  - Stored in `startup.conf` under XDG/HOME config location.
  - Loaded on later launches when `--data-dir` is not provided.
- Added command-line preference logic:
  - `--data-dir` is preferred when provided.
  - If missing/invalid, normal startup falls back to bootstrap flow.
  - Data-required CLI actions fail cleanly when no valid root is available.

## Notes

- Folder selection cancel path is handled gracefully and remains in setup state.
- Validation errors are surfaced clearly via logs and setup dialogs.
- No game rendering/simulation behavior was changed.
