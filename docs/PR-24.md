# PR 24 - Bounded Win95 PE Resource Payload Decoding (VERSION + STRINGTABLE)

## Summary
This PR extends the existing bounded PE/resource reconnaissance pipeline with bounded decoding for selected, low-risk PE resource payloads:
- `VERSION` resources (`VS_VERSION_INFO` root, fixed file info when present, string key/value pairs)
- `STRINGTABLE` resources (16-entry bundles with preserved raw IDs)

`ACCELERATOR` decoding is intentionally out of scope for this PR and is reported as unsupported/skipped.

## What changed
- Added decoded payload models:
  - `PeVersionResource`
  - `PeStringTableResource`
  - `PeResourcePayloadReport`
  - supporting `...Entry`/`...Skip` structs
- Added bounded payload decoders that operate on resource leaves discovered by PR 23:
  - `decode_pe_resource_payloads(...)`
- Added stable formatting helpers:
  - `format_pe_version_resource_report(...)`
  - `format_pe_string_table_report(...)`
  - `format_pe_resource_payload_report(...)`
- Added CLI mode:
  - `--probe-exe-resource-payloads <path>`
- Added tests for:
  - valid minimal VERSION payload decode
  - malformed VERSION payload failure
  - valid STRINGTABLE payload decode
  - malformed STRINGTABLE payload failure
  - stable payload report output

## Guardrails and safety
- Parsing remains bounded and `BinaryReader`-based.
- No disassembly, no execution, no broad generic Win32 resource framework.
- Unsupported resource payload types are explicitly reported as skipped.
- Malformed supported payloads return structured parse errors.
- Raw resource IDs and metadata are preserved in decoded results.
