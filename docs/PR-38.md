# PR 38 - Win95 Runtime Bootstrap Image Path

## Scope

This PR adds the first minimal runtime bootstrap path for Win95 data installs:

- validate or acquire a Win95 data root via existing startup/setup flow
- select one startup ILBM image from real game data
- load the image through existing bounded file loading
- parse ILBM and convert to RGBA through existing decode pipeline
- render the image in the main SDL runtime loop until close or Escape/quit

## Selected startup asset

The bootstrap asset selection order is intentionally narrow and deterministic:

1. `data/forum.lbm`
2. `data/empire2.lbm`
3. `data/fading.lbm`

If an override path is provided in application options, that path is tried first.

## Why this is a bootstrap-only milestone

The purpose of this change is to prove the end-to-end runtime boot path using real Caesar II Win95 assets, without introducing unrelated systems. It validates that Romulus can:

- transition from startup/setup to runtime
- consume real install data with bounded loaders/parsers
- display decoded ILBM output in the runtime loop

This is the smallest practical runtime checkpoint before implementing higher-level runtime systems.

## Explicit non-goals

- gameplay logic
- city/map simulation
- tile atlas or map rendering
- camera systems
- menu/UI framework work
- ImGui integration
- broad scene framework architecture
