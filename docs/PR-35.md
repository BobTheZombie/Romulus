# PR 35 — Win95 PACK bounded batch text-like indexing/export

This PR extends the existing Win95 `PACK` text-like tooling from PR 34 (single-entry extraction) to deterministic bounded **batch analysis + success indexing + export gating**.

## Scope

Added bounded paths for `text-like` entries only:
- batch decode/validation analysis for all `text-like` entries in a PACK
- stable success index generation for decodable text entries
- export commands that only allow entries proven successful by the text success index
- optional bounded first-N batch text export

Out of scope (unchanged):
- PACK structural validation guarantees
- support for any new payload families
- broad archive extraction semantics

## Data-layer additions

New models:
- `Win95PackTextBatchEntryResult`
- `Win95PackTextBatchResult`
- `Win95PackTextIndexEntry`
- `Win95PackTextIndex`

New APIs:
- `analyze_win95_pack_text_batch(...)`
- `build_win95_pack_text_success_index(...)`
- `find_win95_pack_text_index_entry(...)`
- `format_win95_pack_text_batch_report(...)`
- `format_win95_pack_text_index_report(...)`

The batch analyzer reuses PR 34 extraction/validation (`extract_win95_pack_text_entry`) for each `text-like` candidate.

## Behavior

For each `text-like` entry (entry-index order):
1. bounded payload extraction
2. bounded text validation/decode

Per-entry result records:
- index / offset / size / classification hint
- decode success/failure
- compact failure reason (if failed)
- line count / char count / preview (if successful)

Container-level summary includes:
- total entries
- text-like candidate count
- decoded success count
- failed count
- grouped failure reasons (stable order)

## CLI additions

New narrow commands:
- `--probe-pack-text-batch <container>`
- `--probe-pack-text-batch-entries-all`
- `--index-pack-text <container>`
- `--index-pack-text-entries-all`
- `--export-pack-text-success <container> <entry_index> --export-output <path>`
- `--export-pack-text-first <container> <count> --export-output-dir <dir>` (bounded to max 64)

Mode exclusivity remains strict with existing unrelated modes.

## Tests

Added synthetic multi-entry text batch fixture covering:
- successful text-like decodes
- malformed text-like decode failure
- non-text-like entry excluded from text batch

Added assertions for:
- stable batch ordering
- grouped failure summaries
- stable index formatting/truncation
- success-index lookup rules
- export gating behavior (failed text-like entry remains non-exportable)
