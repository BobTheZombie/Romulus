# PR 48: Bounded PL8 sprite-table parser (type-0 decode only)

This PR pivots large-PL8 handling from a single-framebuffer assumption toward a bounded sprite-table interpretation for Caesar II `.PL8` resources.

## Scope

Implemented in this PR:

1. Parse `.PL8` as a sprite-table resource with deterministic bounds checks.
2. Parse file-level header fields:
   - `flags` (u16)
   - `sprite_count` (u16)
   - `reserved`/unknown bytes (u32 raw)
3. Parse sprite descriptors (16 bytes each):
   - `width` (u16)
   - `height` (u16)
   - `data_offset` (u32)
   - `x` (i16)
   - `y` (i16)
   - `tile_type` (u16)
   - `extra_rows` (u16)
4. Decode **type-0** sprites only (`width * height` indexed bytes at `data_offset`).
5. Pair decoded indexed data with external `.256` palette via existing indexed->RGBA conversion paths.
6. Add CLI/reporting for probing sprite tables and viewing/exporting one selected sprite.

## Header interpretation

- Header starts at byte `0`.
- Header size is fixed at `8` bytes.
- `flags & 0x0001` is exposed as `rle_encoded()`.
- Descriptor table starts at byte `8` and contains `sprite_count` entries.

## Supported sprite decode in PR 48

- `tile_type == 0`: supported (simple bitmap decode).
- `tile_type in {1,2,3,4}` or anything else: explicit unsupported failure.
- Header-level RLE flag set: explicit unsupported failure in this PR.

## CLI additions

- `--probe-pl8-sprites <path>`
  - Parses header + descriptors and prints deterministic per-sprite support status.
- `--view-pl8-sprite-pair <imagepl8> <palette256> --sprite-index <n>`
  - Decodes selected type-0 sprite and opens viewer.
- `--export-pl8-sprite-pair <imagepl8> <palette256> --sprite-index <n> --export-output <path>`
  - Decodes selected type-0 sprite and exports image.

## Non-goals

- No generic decode for all PL8 tile types.
- No iso tile decoding framework.
- No gameplay/map/UI semantics for descriptor fields.
- No ImGui integration.

## Failure behavior

Unsupported or malformed inputs fail explicitly and deterministically:

- truncated descriptor table
- out-of-range sprite index
- invalid data offset/size window
- unsupported tile type
- unsupported RLE-flagged decode
