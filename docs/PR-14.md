# PR 14 - Caesar II MAP0 Trailing Undecoded Data Recon/Reporting

PR 14 adds a narrow reconnaissance/reporting layer for undecoded bytes that remain after the known MAP0 terrain and overlay payload sections.

## Scope

- ✅ reuses the existing validated MAP0 header + terrain + overlay parse path
- ✅ records practical decoded section offsets/sizes
- ✅ preserves acceptance of trailing undecoded bytes
- ✅ adds stable report output for trailing undecoded data
- ❌ does not assign semantics to trailing data
- ❌ does not add rendering or simulation logic

## Changes

### 1) `MapResource` decode metadata

Added bounded bookkeeping fields to report known section layout:

- `terrain_payload_offset`, `terrain_payload_size`
- `overlay_payload_offset`, `overlay_payload_size`
- `trailing_data_offset`, `trailing_data_size`
- `trailing_data_preview` (fixed-size leading preview for recon)

### 2) Parse path updates (still BinaryReader-based)

`parse_caesar2_map` now:

1. decodes terrain payload from the known header offset
2. decodes overlay payload from the computed post-terrain offset
3. records `reader.tell()` and `reader.remaining()` as trailing undecoded offset/size
4. reads a bounded preview window (`kTrailingPreviewSize`, currently 16 bytes) from trailing data

No chunk semantics are inferred from trailing bytes.

### 3) Report updates

`format_map_report` now includes:

- decoded terrain/overlay ranges (offset + size)
- trailing undecoded range (offset + size)
- trailing preview line with a compact hex sample (`0xNN` tokens)

This output is intentionally stable and RE-friendly.

### 4) Tests

Extended map tests to cover:

- payload offset/size bookkeeping for terrain + overlay
- trailing undecoded offset/size and preview capture
- capped trailing preview behavior for large trailing regions
- report output lines for new range and trailing preview fields

## Definition of done mapping

- MAP0 reports undecoded bytes after known sections: ✅
- Report output is stable/useful for reverse engineering: ✅
- Tests cover trailing-data reporting behavior: ✅
