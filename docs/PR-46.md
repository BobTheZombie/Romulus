# PR 46: Bounded RAT_BACK structured large-PL8 parsing

## Summary
This PR adds bounded support for a second large-PL8 variant used by `DATA0/RAT_BACK.PL8` while leaving the existing FORUM parser unchanged.

Implemented scope:
- Added a dedicated bounded parser/model for RAT_BACK-style structured PL8 payloads after the shared 24-byte header.
- Added deterministic structured-prefix parsing as little-endian 32-bit fields with raw value reporting.
- Added deterministic candidate-region evaluation for primary indexed image isolation.
- Added structured report formatting for reverse-engineering.
- Added RAT_BACK structured pair decode path (`.PL8` + `.256`) that reuses the existing indexed-to-RGBA conversion logic.
- Added CLI commands:
  - `--probe-pl8-structured <path>`
  - `--view-pl8-structured-pair <imagepl8> <palette256>`
  - `--export-pl8-structured-pair <imagepl8> <palette256> --export-output <path>`
- Updated forum composition overlay decode to attempt FORUM parser first, then RAT_BACK structured parser fallback.
- Added tests for structured success/failure/report/decode scenarios.

## Bounded parsing behavior
The RAT_BACK parser:
1. Parses the confirmed shared 24-byte outer header (dimensions at offset 8).
2. Requires at least a 16-byte structured payload prefix.
3. Reads 4 x `u32le` prefix fields and stores them as raw fields.
4. Evaluates deterministic candidate regions using all `(offset_field, size_field)` combinations.
5. Accepts candidates only when all bounds checks pass and size exactly matches `width * height`.
6. Requires exactly one accepted candidate; otherwise fails cleanly with a precise reason.

No gameplay/UI semantics are assigned to unknown prefix fields in this PR.

## Notes
- This PR intentionally supports exactly one additional structured variant (RAT_BACK-style), not a generic PL8 framework.
- Failure is explicit and deterministic when no single bounded image region can be isolated.
