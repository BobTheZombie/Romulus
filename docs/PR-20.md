# PR 20: Win95 .PL8 bounded parser + reconnaissance path

## Scope
PR 20 adds a narrow, bounded `.PL8` parser/recon path for one Win95 Caesar II layout only.

Supported layout in this PR:
- exactly **256 RGB triplets** (`256 * 3 = 768` bytes)
- no headers/variants beyond this fixed payload are claimed

Everything else fails explicitly as unsupported layout.

## What was added

### 1) Data-layer `.PL8` resource type
- Added `Pl8Resource` with decoded palette entries and payload offsets/sizes.
- Added parser entry points:
  - `parse_pl8_resource(std::span<const std::byte>)`
  - `parse_pl8_resource(std::span<const std::uint8_t>)`

### 2) Bounded parser for one supported structure
- Parser is BinaryReader-based and bounded.
- Validates exact size (`768` bytes).
- Decodes exactly 256 RGB entries.
- Uses explicit invalid-format errors for unsupported layout sizes.

### 3) Stable report helper
- Added `format_pl8_report(...)` for deterministic reverse-engineering/debug output.
- Includes a fixed report header, layout label, payload metadata, and a capped palette preview.

### 4) CLI probe support for one `.PL8` file
- Added `--probe-pl8 <path>` mode in `apps/caesar2/main.cpp`.
- Mode is mutually exclusive with other command modes.
- Loads one file and prints stable PL8 report output.

### 5) Tests
- Added `tests/data/pl8_resource_tests.cpp`:
  - valid parse for supported fixed-size layout
  - malformed/unsupported size rejection
  - stable report text assertions

## Constraints honored
- Parsing remains bounded and BinaryReader-based.
- Unsupported layouts fail explicitly; no speculative decoding.
- Scope remains narrow and focused on Win95/C2Win95 reconnaissance.
- No broad rendering/resource-manager architecture added.
- No simulation logic added.
