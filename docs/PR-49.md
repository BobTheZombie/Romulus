# PR-49: Bounded multi-sprite PL8 decode/composition for type-0 sprite tables

## Scope delivered

This PR adds bounded decoding and composition for Caesar II Win95 PL8 sprite-table resources where sprites are:

- file-header non-RLE (`flags & 1 == 0`), and
- descriptor `tile_type == 0`.

Decoded sprite indexed pixels are paired with `.256` palettes and converted through the existing indexed-to-RGBA conversion path.

## Composition model

- Parse sprite-table descriptors with existing bounded parser.
- Decode each sprite in descriptor order.
- Build composition bounds from descriptor coordinates (`x`, `y`) and dimensions (`width`, `height`).
- Allocate a transparent RGBA canvas bounded to deterministic maximum dimensions.
- Blit sprites in descriptor order onto the canvas.
- Treat palette index `0` as transparent when decode is requested with transparent-zero semantics (matching overlay composition behavior).

This makes `RAT_BACK.PL8 + RAT_BACK.256` composable as a sprite-table overlay layer.

## Runtime integration

Forum composition now attempts overlays in this order:

1. FORUM-style PL8 decode path
2. multi-sprite sprite-table decode/composition path (this PR)
3. existing RAT_BACK structured fallback path

On failure, decode paths still skip cleanly with deterministic logs and continue processing remaining overlays.

## CLI behavior

The following commands now use the multi-sprite composition path for supported sprite tables:

- `--view-pl8-sprite-pair <imagepl8> <palette256>`
- `--export-pl8-sprite-pair <imagepl8> <palette256> --export-output <path>`

`--probe-pl8-sprites <path>` remains available for descriptor/header probing.

## Reporting additions

Sprite-pair report output now includes:

- whether full sprite decode is supported
- composed canvas bounds and dimensions when composition occurs
- per-sprite decode/composition status

Output is compact and deterministic.

## Explicit non-goals

- No ISO tile decode support
- No RLE sprite decode support
- No generic PL8 framework refactor beyond current sprite-table model
- No gameplay systems or UI-system feature work
