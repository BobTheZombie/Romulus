# PR 21 - Win95/C2Win95 .PL8 Batch Reconnaissance

## Summary
This PR adds a narrowly scoped batch reconnaissance path for Caesar II Win95 `.PL8` assets.

It does **not** attempt broad `.PL8` decoding semantics. It only distinguishes:
- supported palette-style layout: exactly 768-byte payload (256 RGB triplets)
- unsupported/other layouts

## What changed
- Added `probe_pl8_files(...)` to inspect multiple selected `.PL8` files in one invocation.
- Added `format_pl8_batch_report(...)` for deterministic, compact batch reporting.
- Kept unsupported layouts explicit as unsupported summaries, rather than treating them as malformed by default.
- Reused the existing bounded fixed-layout parser for the supported 768-byte case.
- Extended CLI `--probe-pl8` handling to accept multiple occurrences in one run.
- Added deterministic tests for supported/unsupported batch formatting behavior.

## Output shape
For each requested file, the batch report includes:
- `path`
- `size_bytes`
- `matches_supported_palette_layout` (`yes`/`no`)
- `palette_summary` for supported layout files
- `unsupported_summary` for unsupported/other layouts

The output order follows invocation order and is deterministic.

## Scope guardrails
- No rendering/simulation logic added.
- No claims of supporting all `.PL8` variants.
- No attempt to decode unsupported layouts beyond concise layout classification.
