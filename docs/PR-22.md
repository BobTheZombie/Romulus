# PR 22 - Bounded Win95 PE Executable Reconnaissance

## Summary
This PR adds a bounded PE32 reconnaissance layer for Win95 executables (such as `CAESAR2.EXE`) focused on metadata and import table extraction.

It is intentionally **reconnaissance-only**:
- no disassembly
- no code execution
- no control-flow analysis
- no gameplay semantic inference

## What changed
- Added `PeExeResource` and parser/report APIs:
  - `parse_pe_exe_resource(...)`
  - `format_pe_exe_report(...)`
- Added bounded validation and decoding for:
  - DOS header (`MZ`)
  - PE signature (`PE\0\0`)
  - `IMAGE_FILE_HEADER`
  - bounded `IMAGE_OPTIONAL_HEADER` subset (PE32 only)
- Added explicit PE32+ rejection and structured malformed-input failures.
- Added bounded import table extraction:
  - DLL names
  - named imported functions
  - ordinal-only imports are safely ignored when unnamed
- Added deterministic categorized import summary sections for:
  - file I/O related imports
  - windowing/UI imports
  - graphics/palette imports
  - audio/video imports
- Added CLI support for executable probing:
  - `--probe-exe <path>`
- Enforced `--probe-exe` mutual exclusion with existing probe/export/view modes.
- Added synthetic fixture tests covering valid parse, signature failures, truncated imports, import extraction, and stable report text.

## Output shape
The PE report includes:
- image base
- entry point RVA
- subsystem
- section count
- timestamp
- has-resources / has-relocations flags
- import descriptors and named functions
- compact categorized import summary

## Scope guardrails
- Bounded BinaryReader-based parsing only.
- No OS PE parsing APIs.
- No third-party PE libraries.
- No disassembly or execution paths introduced.
