# PR 25 - Enriched PE EXE Report with Decoded Resource Payload Summaries

This PR integrates bounded decoded Win95 PE resource payload data into the main EXE probe report path, so `--probe-exe <path>` can surface compact VERSION/STRINGTABLE insights in a single pass.

## What changed

- Extended PE EXE report formatting with an enriched path that accepts decoded payload data:
  - new overload: `format_pe_exe_report(const PeExeResource&, const PeResourcePayloadReport&)`
- Added compact payload summaries to the main EXE report when decoded payloads are available:
  - VERSION summary
    - file/product versions (when fixed file info exists)
    - key string fields: `CompanyName`, `FileDescription`, `ProductName`
  - STRINGTABLE summary
    - decoded string entry count
    - bundle ID coverage summary
    - decoded string ID ranges
    - bounded preview (up to 6 strings)
  - skipped/unsupported payload summary
    - total skipped payload count
    - compact per-type counts
- Wired `--probe-exe <path>` to attempt payload decoding and print the enriched EXE report when decoding succeeds.
- Preserved dedicated detailed report helpers from PR 24 (`format_pe_version_resource_report`, `format_pe_string_table_report`, and `format_pe_resource_payload_report`) unchanged.

## Scope constraints honored

- No new resource payload decoders were introduced.
- No disassembly, execution, or control-flow analysis was added.
- No broad reporting framework refactor was introduced.
- Output remains bounded and deterministic (sorted/ranged summaries + fixed preview cap).

## Tests

Added tests in `tests/data/pe_exe_resource_tests.cpp` covering:

- enriched EXE report VERSION summary output
- enriched EXE report STRINGTABLE summary output
- deterministic/stable formatting behavior
- skipped payload per-type summary behavior
