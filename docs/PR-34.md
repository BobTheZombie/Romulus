# PR 34 — Win95 PACK bounded text-like extraction

## Summary
This PR adds a **single new PACK entry family** on top of the existing bounded parser/analyzer stack: `text-like`.

Scope remains intentionally narrow and deterministic:
- no generic extractor framework
- no new gameplay semantics
- no changes to PACK structural validation guarantees from PR 29
- no weakening of bounded candidate analysis behavior from PR 30

## Added bounded text extraction path
A new helper extracts a PACK entry by index only when the entry classification is exactly `text-like`:
- `extract_win95_pack_text_entry(...)`

It reuses existing parsed PACK metadata and enforces:
1. entry index bounds checks
2. overflow-safe `offset + size` validation
3. extraction bounds checks against container byte span
4. explicit family gating (`text-like` only)
5. bounded text decode validation (ASCII-printable plus `\n`, `\r`, `\t`)

On success, extraction returns:
- copied raw payload bytes
- decoded text payload
- deterministic `line_count` and `character_count`

On failure, extraction returns explicit, structured parse errors (invalid index, unsupported family, out-of-bounds, or text validation failure).

## Stable text report helper
Added:
- `format_win95_pack_text_report(...)`
- `Win95PackTextReportOptions` with bounded `preview_character_limit`

Report includes:
- entry index
- offset
- size
- classification
- line/character counts
- deterministic escaped preview (`\n`, `\r`, `\t` rendered visibly)
- explicit truncation marker (`text_preview_truncated: yes|no`)

## CLI support
Added narrow CLI modes:
- `--probe-pack-text <container> <entry_index>`
- `--export-pack-text <container> <entry_index> --export-output <path>`

Behavior:
- uses the bounded text extraction helper only
- enforces strict entry-index parsing
- enforces strict mode mutual exclusivity with other command families
- requires `--export-output` for text export
- writes decoded text bytes deterministically to output file

## Tests
`tests/data/win95_data_container_tests.cpp` now covers:
- valid synthetic PACK fixture with text-like payload
- invalid entry index rejection
- non-text-like family rejection
- truncated payload bounds failure
- malformed/non-decodable text rejection
- stable report/preview formatting (with explicit truncation)
- valid text export path behavior
