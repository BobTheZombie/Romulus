# PR-41: Bounded Win95 `.256` + `.PL8` overlay decode path

## Scope
This PR adds a narrow, bounded decode path for Caesar II Win95 screen assets stored as paired files:

- `*.256`: flat indexed pixel payload (no header)
- `*.PL8`: 256-entry RGB palette in the already-supported 768-byte layout

The implementation introduces:

1. A bounded `.256` parser/resource model with explicit dimensions.
2. A narrow paired `.256 + .PL8` decode helper.
3. CLI probe/view/export commands for this pair format.
4. Focused tests for parser, pair decode, failure handling, and stable reporting.

## Why `.256 + .PL8` is treated as a second rendering system
Romulus already supports ILBM/PBM (`.LBM`) startup/background assets, but forum/rating overlays appear to be provided separately as paired indexed + palette resources.

Those assets are not ILBM container files and therefore require a distinct decode path while still reusing existing indexed-image conversion and RGBA export/view infrastructure.

## Explicit dimension handling
`.256` files in this scoped implementation are treated as raw indexed payloads with **no embedded dimensions**.

- The low-level parser requires caller-supplied `width` + `height`.
- The parser validates strict `payload_size == width * height`.
- No generic dimension guessing is performed in the parser.

For bounded bootstrap/debug convenience, a narrow helper provides known dimensions for initial targets:

- `FORUM.256` -> `640x480`
- `RAT_BACK.256` -> `640x480`

CLI commands can pass dimensions explicitly via `--width` and `--height`, and if omitted, only these known mappings are used.

## CLI additions
- `--probe-256 <path> [--width <w> --height <h>]`
- `--view-256-pl8 <image256> <palettepl8> [--width <w> --height <h>]`
- `--export-256-pl8 <image256> <palettepl8> --export-output <path> [--width <w> --height <h>]`

Mutual exclusivity remains strict with unrelated command families.

## Non-goals
- No gameplay systems.
- No map rendering.
- No ImGui integration.
- No broad generic asset manager.
- No automatic composition with `FORUM.LBM` background in this PR.

This PR is limited to bounded decoding and direct visualization/export of one `.256 + .PL8` overlay asset path.
