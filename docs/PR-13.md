# PR 13 - Caesar II MAP0 Overlay Tile Payload Decoding

## Summary
PR 13 extends MAP0 parsing to decode the overlay tile payload section that follows the terrain payload, while preserving bounded, section-oriented parsing behavior.

This PR remains intentionally narrow:
- ✅ decodes raw overlay tile bytes using header-derived overlay count
- ✅ preserves strict bounds checks and structured parse failures
- ✅ preserves widened arithmetic for payload math
- ✅ allows trailing undecoded bytes after overlay payload
- ❌ does not interpret overlay semantics
- ❌ does not render maps
- ❌ does not add simulation/gameplay logic

## What was added

### 1) `MapResource` overlay storage
- Added `overlay_tiles` (`std::vector<std::uint8_t>`) to hold raw overlay payload bytes.
- Overlay values remain uninterpreted in this PR.

### 2) Overlay section decoding in `parse_caesar2_map(...)`
- Reused the existing validated header parse path.
- Kept terrain and overlay responsibilities separated by decoding each payload section independently.
- Parsing flow now:
  1. parse/validate MAP0 header
  2. decode terrain payload from `kHeaderSize` using `terrain_tile_count`
  3. decode overlay payload immediately after terrain using `overlay_tile_count`
- All decoding uses `BinaryReader` reads.
- Truncated terrain or overlay payloads fail cleanly with structured parse errors.
- Trailing undecoded bytes are still accepted.

### 3) Report helper updates
- `format_map_report(...)` now includes:
  - overlay decoded byte-count summary
  - a small overlay sample window (mirroring terrain sample behavior)

### 4) Tests
Added/updated tests cover:
- valid map decode including overlay payload
- truncated overlay payload failure with the expected offset
- report output including overlay summary/sample
- existing terrain decode and failure behavior remains covered

## Safety notes
- Payload size and offset calculations use widened arithmetic before narrowing.
- Bounds checks remain `BinaryReader`-driven for section reads.
- Errors preserve offset/code/message context for malformed inputs.
