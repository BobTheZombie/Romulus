# PR-52: PL8 sprite-index support for view/export

## Summary
PR-52 adds optional `--sprite-index <n>` support to PL8 sprite-table view/export flows:

- `--view-pl8-sprite-pair <imagepl8> <palette256> [--sprite-index <n>]`
- `--export-pl8-sprite-pair <imagepl8> <palette256> --export-output <path> [--sprite-index <n>]`

This preserves existing composed multi-sprite output when no sprite index is provided.

## Behavior

### Without `--sprite-index`
- Decode all supported sprites from the PL8 sprite table.
- Reuse existing composition behavior and render/export the composed forum-canvas output.

### With `--sprite-index <n>`
- Decode PL8 sprite table and palette using the same existing decode path.
- Validate sprite index against descriptor table bounds.
- Render/export only the selected decoded sprite.
- Output image dimensions are the selected sprite dimensions (`width` x `height`).
- Index-0 transparency behavior is unchanged and still controlled by `--index-zero-transparent`.

## Error handling
Explicit failures are emitted for:
- sprite index outside sprite-table bounds
- selected sprite decode unsupported/failed (for example unsupported tile type)
- export/view failures after decode

## Logging/reporting
Added compact logs to indicate output mode:
- composed multi-sprite output
- or single selected sprite output with index and dimensions

## Non-goals
- No changes to PL8 sprite-table parsing rules
- No changes to sprite composition semantics
- No new parser families
- No changes to structured/image PL8 pair workflows
