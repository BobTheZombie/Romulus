# PR 30 - Bounded PACK entry-table analysis and preview reporting

## Summary

PR 30 extends the Win95 `PACK` reconnaissance pipeline from PR 29 by adding bounded per-entry analysis and compact preview reporting.

This remains **analysis-only**:

- no payload extraction
- no recursive decoding
- no gameplay semantics

## New entry-level analysis

After PR 29 structural validation succeeds, each entry now includes bounded metadata:

- entry index, offset, size, end offset
- fixed-width leading signature bytes (hex)
- printable ASCII signature preview
- coarse bounded hint:
  - `possible-ilbm`
  - `text-like`
  - `opaque-binary`
  - `empty`
- bounded recognizable-signature flag (heuristic only)

Heuristics are explicit and conservative; unknown binary data remains `opaque-binary`.

## Container summary

The PACK resource now includes a deterministic summary across entries:

- entry count
- total payload bytes
- recognizable-signature count
- size distribution buckets (`0-255`, `256-4095`, `4096-65535`, `65536+`)
- signature/magic frequency hints (top bounded set)

## Reporting changes

`format_win95_data_container_report(...)` now emits:

- stable container header metadata
- stable summary block
- bounded `entries_preview` section

Default report output is compact (`8` entries). Truncation is explicit. Extended mode can include all entries.

## CLI

`--probe-win95-container <path>` now includes compact bounded entry previews by default.

Added optional switch:

- `--probe-win95-container-entries-all`

This flag requires `--probe-win95-container` and enables full entry preview listing (still analysis-only).

## Tests

Extended `romulus_win95_data_container_tests` coverage for:

- synthetic multi-entry signature/classification analysis
- bounded preview formatting + truncation marker
- deterministic summary/report sections
- extended preview mode includes all entries
- malformed/truncated payload range failure behavior
