# PR-55: Restore mixed semantic anchor mapping for RAT_BACK.PL8

## Summary
This PR restores explicit mixed anchor semantics for `RAT_BACK.PL8` placement. The prior uniform `BottomCenter` override from PR-54 was reverted for `sprite[0]` because runtime validation showed the tall column being pushed off-screen.

## Why uniform BottomCenter was incorrect
`RAT_BACK.PL8` is a mixed-semantic sprite table:
- `sprite[0]` (`96x337`) is a tall column that must anchor from the top to remain visible.
- `sprite[1]` (`96x24`) is lower trim and aligns correctly when bottom-anchored.
- `sprite[2]` (`96x30`) is the base and also aligns correctly when bottom-anchored.

A uniform `BottomCenter` rule incorrectly treated the column with the same vertical anchor as the trim/base elements, causing incorrect placement.

## Final mapping (explicit and deterministic)
For `RAT_BACK.PL8` only:
- `sprite[0]` -> `TopCenter`
- `sprite[1]` -> `BottomCenter`
- `sprite[2]` -> `BottomCenter`

Fallback behavior for non-RAT_BACK assets is unchanged and still uses the caller-provided placement mode.

## Placement math (unchanged)
No placement formulas changed in this PR:
- `TopCenter`: `dest_x = x - (width / 2)`, `dest_y = y`
- `BottomCenter`: `dest_x = x - (width / 2)`, `dest_y = y - height`

## Logging / debug behavior
Composition debug reporting remains deterministic and continues to emit, per sprite:
- sprite index
- resolved placement mode
- final destination rectangle

## Non-goals
- No PL8 parser changes
- No decode or extraction changes
- No palette pairing changes
- No composition order changes
- No new placement modes
- No heuristics or inference
