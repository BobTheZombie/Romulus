# PR 11 - Caesar II Map Header Parsing (Initial Reverse-Engineering Step)

## Summary
PR 11 introduces a narrow, safe parser for one known Caesar II map header layout and a report helper for debug inspection.

This PR is intentionally limited to header extraction and validation. It does **not** add map rendering, simulation, or gameplay logic.

## What was added

### 1) Map resource type
- Added `MapResource` as a structured representation of the parsed map header.
- Captures:
  - `format_version`
  - `header_size`
  - `width`
  - `height`
  - `terrain_tile_count`
  - `overlay_tile_count`
  - `flags`
  - `random_seed`

### 2) Header parser
- Added `parse_caesar2_map_header(...)` overloads for `std::span<const std::byte>` and `std::span<const std::uint8_t>`.
- Parsing uses `BinaryReader` exclusively.
- Validates:
  - magic (`MAP0`)
  - known header size (`28` bytes)
  - non-zero, bounded dimensions
  - terrain tile count matches `width * height`
  - overlay tile count does not exceed terrain tile count
- Malformed inputs fail cleanly via `ParseError`.

### 3) Debug/report helper
- Added `format_map_report(...)` to emit a concise structural report for parsed map headers.

### 4) Tests
- Added positive and negative tests covering:
  - valid header parse
  - invalid magic
  - malformed tile counts
  - overlay count bounds violation
  - truncated header / out-of-bounds behavior
  - report formatting

## Scope notes
- Header-only parsing.
- No map payload decoding.
- No rendering.
- No simulation logic.
