# PR 32 - Bounded batch ILBM indexing from Win95 PACK containers

## Scope
PR 32 extends PR 31's single-entry extraction path into a deterministic batch analysis mode for entries already classified as `possible-ilbm`.

Implemented capabilities:
- analyze every PACK entry flagged as `possible-ilbm`
- reuse bounded single-entry extraction + ILBM validation from PR 31
- capture deterministic per-entry success/failure records
- summarize container-level parse totals and grouped failure reasons
- expose compact/detailed stable batch reporting
- add CLI mode for bounded batch probing

## Batch analysis model
New result models:
- `Win95PackIlbmBatchResult`
- `Win95PackIlbmBatchEntryResult`

Per analyzed candidate entry, the batch result records:
- entry index
- offset and size
- classification hint
- parse success/failure state
- compact failure reason (for failures)
- ILBM summary fields (`width`, `height`, `palette_color_count`) when parsing succeeds

Container summary includes:
- total PACK entries
- `possible-ilbm` candidate count
- successful ILBM parse count
- failed parse count
- grouped failure-reason frequencies

## Reporting behavior
`format_win95_pack_ilbm_batch_report(...)` provides deterministic output:
- stable title and summary fields
- grouped failure reason section
- bounded preview by default
- optional all-entries mode (`include_all_entries=true`)

## CLI
New command mode:
- `--probe-pack-ilbm-batch <container>`

Optional detailed listing mode:
- `--probe-pack-ilbm-batch-entries-all`

Validation rules enforced:
- batch mode is mutually exclusive with other PACK ILBM modes
- `--probe-pack-ilbm-batch-entries-all` requires batch mode

## Non-goals preserved
This PR intentionally does **not**:
- add any new PACK payload families
- build a generalized extraction framework
- infer gameplay semantics
- relax PR 29 structural PACK validation
- weaken PR 30 bounded analysis behavior
