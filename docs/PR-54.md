# PR-54: Uniform BottomCenter semantics for RAT_BACK sprite placement

## Summary
This PR corrects `RAT_BACK.PL8` semantic placement by removing the mixed per-sprite anchor model and applying a single deterministic anchor rule to all known RAT_BACK sprites.

## Why the previous mixed policy was incorrect
The previous mapping used:
- `sprite[0]` -> `TopCenter`
- `sprite[1]` -> `BottomCenter`
- `sprite[2]` -> `BottomCenter`

Runtime composition results showed the tall column (`sprite[0]`) vertically detached from the lower trim/base pieces even though horizontal centering was correct. That behavior is consistent with using the wrong vertical anchor for `sprite[0]`.

## Corrected semantic rule
For `RAT_BACK.PL8`, all known sprites now resolve to `BottomCenter`:
- `sprite[0]` -> `BottomCenter`
- `sprite[1]` -> `BottomCenter`
- `sprite[2]` -> `BottomCenter`

Placement math is unchanged and continues to use existing `BottomCenter` behavior:
- `dest_x = x - (width / 2)`
- `dest_y = y - height`

## Logging / determinism
Composition-time placement debug output remains deterministic and now reports the resolved `BottomCenter` mode uniformly for RAT_BACK sprite debug entries.

## Non-goals
- No changes to PL8 parser behavior
- No changes to PL8 decode or sprite extraction
- No changes to forum composition order
- No new placement modes
- No new heuristics or auto-detection
