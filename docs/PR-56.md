# PR-56: Bounded RAT_FRON sprite-table decode + inspection

## Scope

This PR keeps the existing PL8 sprite-table/type-0 parser family intact and extends inspection ergonomics so `RAT_FRON.PL8 + RAT_FRON.256` can be assessed through the same deterministic path already used for `RAT_BACK` sprite-table work.

### What changed

- Reused the existing bounded sprite-table parser (`parse_caesar2_pl8_sprite_table`) and type-0 decode path (`decode_caesar2_pl8_type0_sprite`) without adding a new parser family.
- Expanded sprite-pair report formatting to include image header fields and per-sprite descriptor fields (`width/height/x/y/tile_type/extra_rows`) plus decode/composition status.
- Added single-sprite probe reporting (`--probe-pl8-sprites ... --sprite-index <n>`) for deterministic per-sprite inspection.
- Preserved existing view/export commands for sprite-pair decode:
  - `--view-pl8-sprite-pair <imagepl8> <palette256> [--sprite-index <n>]`
  - `--export-pl8-sprite-pair <imagepl8> <palette256> --export-output <path> [--sprite-index <n>]`

## RAT_FRON model fit

`RAT_FRON.PL8` is now evaluated strictly through the current bounded sprite-table/type-0 support model:

- If the file parses as a sprite-table and sprites are type-0 with in-bounds payload windows, decode is reported as `supported_type0`.
- If it fails this model (e.g. RLE flag, unsupported tile type, invalid payload windows), failure is explicit and deterministic in probe/decode reports (`unsupported_rle`, `unsupported_tile_type`, `invalid_offset_or_size`, or structured parse error text).

No speculative parser broadening is introduced in this PR.

## Inspection commands

Use these commands against `DATA0/RAT_FRON.PL8` + `DATA0/RAT_FRON.256`:

```bash
./caesar2 --probe-pl8-sprites DATA0/RAT_FRON.PL8
./caesar2 --probe-pl8-sprites DATA0/RAT_FRON.PL8 --sprite-index 0
./caesar2 --view-pl8-sprite-pair DATA0/RAT_FRON.PL8 DATA0/RAT_FRON.256
./caesar2 --view-pl8-sprite-pair DATA0/RAT_FRON.PL8 DATA0/RAT_FRON.256 --sprite-index 0
./caesar2 --export-pl8-sprite-pair DATA0/RAT_FRON.PL8 DATA0/RAT_FRON.256 --export-output out/rat_fron.ppm
./caesar2 --export-pl8-sprite-pair DATA0/RAT_FRON.PL8 DATA0/RAT_FRON.256 --sprite-index 0 --export-output out/rat_fron_sprite0.ppm
```

## Non-goals

- No new PL8 parser family.
- No changes to FORUM image decode path.
- No changes to RAT_BACK structured parser path.
- No UI/layout/gameplay/map rendering additions.
- No speculative runtime forum composition changes unless asset usage becomes obvious.
