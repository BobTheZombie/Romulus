# PR 29 - First bounded Win95 DATA/DATA0 outer container parser

## Summary

PR 29 introduces the first structural parser for a confirmed DATA/DATA0 outer layout: a `PACK`-signature container with a fixed header and a bounded index table.

This PR intentionally stops at container-level structure:

- header signature and entry count
- fixed-size entry table
- per-entry offset/size ranges
- overlap and bounds validation

No embedded payload decoding or gameplay semantics are introduced.

## Confirmed outer layout implemented

`Win95PackContainerResource` now models and parses:

- signature (`PACK`)
- entry count (`u32` little-endian)
- entry table offset (`8`)
- entry table size (`entry_count * 8`)
- entries with `offset`, `size`, and computed `end_offset`

Supported parser API:

- `parse_win95_pack_container(...)`
- `probe_win95_data_container_file(...)`
- `format_win95_data_container_report(...)`

## Safety and validation

The parser is fully bounded and explicit about unsupported/invalid input:

- strict signature check (`PACK` only)
- entry-table size overflow checks
- entry-table-in-file-bounds checks
- per-entry end-offset overflow checks
- per-entry in-file-bounds checks
- rejects entries that point into header/table region
- rejects overlapping non-empty entry ranges

Failures return structured `ParseError` data with stable offsets/messages.

## CLI

Added a narrow CLI mode for container probing:

- `--probe-win95-container <path>`

Behavior:

- uses existing data-root startup validation flow
- mutually exclusive with other probe/export/view modes
- loads file with bounded loader and prints stable container report

## Tests

New `romulus_win95_data_container_tests` coverage includes:

- valid synthetic PACK container parse
- malformed signature failure
- truncated entry-table failure
- invalid offset/header-range failure
- stable deterministic report formatting
