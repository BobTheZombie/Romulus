# PR-53: Deterministic semantic placement for RAT_BACK PL8 sprite-table overlays

## Summary
This PR adds an explicit semantic placement layer for PL8 sprite-table overlays so that known multi-part assets can be placed correctly without changing decode/parser behavior.

The core distinction is:
- **Descriptor placement**: raw `(x, y, width, height)` values decoded from PL8 sprite descriptors.
- **Semantic placement**: deterministic interpretation of those descriptor anchors for specific assets/sprites.

## Why semantic placement is needed
`RAT_BACK.PL8` is not authored as a simple top-left anchored sprite set. It is a composed decorative object with parts that share center/bottom relationships:

- `sprite[0]` (column, `96x337`) uses **TopCenter**.
- `sprite[1]` (trim, `96x24`) uses **BottomCenter**.
- `sprite[2]` (base, `96x30`) uses **BottomCenter**.

Applying top-left semantics to all parts causes visible misalignment.

## Placement modes in this PR
This PR extends `SpritePlacementMode` with:
- `TopCenter`
- `BottomCenter` (retained and used semantically)

Deterministic integer math is used:

- `TopCenter`: `dest_x = x - (width / 2)`, `dest_y = y`
- `BottomCenter`: `dest_x = x - (width / 2)`, `dest_y = y - height`

## Explicit mapping layer
A bounded resolver is introduced:

- `resolve_sprite_placement_mode(asset_name, sprite_index, fallback_mode)`

Current explicit mapping:
- `rat_back.pl8`, `sprite 0` -> `TopCenter`
- `rat_back.pl8`, `sprite 1` -> `BottomCenter`
- `rat_back.pl8`, `sprite 2` -> `BottomCenter`

All non-mapped assets/sprites return the passed fallback mode unchanged.

## Integration details
The composition path now resolves per-sprite placement mode from:
- asset identifier (image PL8 logical path / filename)
- sprite index
- caller fallback mode

Resolved mode is then applied to compute the destination rectangle before blitting.

Debug placement reporting now includes:
- resolved mode per sprite
- final destination rect after semantic placement

## Why this is not a parser change
No PL8 parser, descriptor decode, or sprite extraction semantics were modified. This PR only adjusts **how decoded descriptors are interpreted at composition time** for a specific known asset mapping.

## Non-goals
- No heuristic or inferred mapping.
- No auto-detection of placement semantics.
- No change to forum composition order.
- No generalized semantic inference system (yet).
