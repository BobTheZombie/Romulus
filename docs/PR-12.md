# PR 12 - Caesar II MAP0 Terrain Grid Decoding

## Summary
PR 12 extends map parsing beyond the MAP0 header to decode the terrain tile payload safely using the previously validated header metadata.

This PR remains intentionally narrow:
- ✅ decodes terrain tile bytes
- ✅ validates header tile count against dimensions
- ✅ enforces strict bounds checks and clean parse failures
- ✅ allows trailing undecoded bytes after terrain payload
- ❌ does not decode overlays
- ❌ does not render maps
- ❌ does not add simulation/gameplay logic

## What was added

### 1) `MapResource` terrain storage
- Added `terrain_tiles` (`std::vector<std::uint8_t>`) to hold raw terrain grid values.
- No semantic naming/interpretation is applied yet.

### 2) Full map parser
- Added `parse_caesar2_map(...)` overloads for byte and uint8 spans.
- Parsing flow:
  1. parse and validate header with existing rules
  2. compute terrain payload size with widened arithmetic
  3. seek to payload offset and decode exactly `terrain_tile_count` bytes via `BinaryReader`
- Truncated payloads fail with structured parse errors.
- Trailing bytes are intentionally accepted.

### 3) Report helper updates
- `format_map_report(...)` now includes:
  - terrain decode byte-count summary
  - a small terrain sample window for debugging

### 4) Tests
Added/updated tests cover:
- valid terrain decode
- truncated terrain payload failure
- header tile-count mismatch failure
- trailing undecoded bytes acceptance
- report output content for terrain summary/sample

## Safety notes
- Tile count and payload size calculations use widened arithmetic.
- Bounds checks are retained through `BinaryReader`-driven reads.
- Failures return structured `ParseError` information with offsets.
