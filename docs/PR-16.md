# PR-16: Evidence-Based Signature Registry

## Goal
Add a bounded, deterministic, evidence-based signature registry layer that consumes existing candidate probe output and produces ranked decoder candidates with explicit evidence, scores, and confidence buckets.

## Scope
- Add a signature registry module that reads `CandidateProbeBundleReport` + `CandidateFileReport` only.
- Score candidate decoder matches using deterministic integer weights.
- Produce ranked matches with per-match evidence records.
- Add CLI mode `--match-signature <path>` that probes one candidate and prints a signature match report.
- Add tests for ranking, fallback behavior, pairing boosts, deterministic ordering, and boundedness discipline.

## Non-goals
- No decoder dispatch.
- No semantic parsing.
- No plugin registration.
- No recursive directory crawling.
- No nondeterministic heuristics.

## Evidence kinds
- `ExtensionHint`
- `MagicBytes`
- `StructuralHint`
- `PairingHint`
- `TextPatternHint`
- `FallbackHint`

## Decoder scoring behavior
The initial rule set is additive with explicit evidence:

- `DosExecutable`
  - MZ magic: +80
  - `.exe` extension: +15
  - `mz-executable-like`: +10

- `IniLikeText`
  - `.ini` extension: +20
  - `.cfg` extension: +15
  - `text-like`: +20
  - `[section]` line pattern: +20
  - `key=value` line pattern: +20

- `PlainText`
  - `.txt` extension: +15
  - `text-like`: +30
  - non-empty printable preview: +10

- `Caesar2DatContainer`
  - `.dat` extension: +15
  - matching `.ix` pair present: +35
  - `opaque-binary`: +10

- `Caesar2IxIndex`
  - `.ix` extension: +20
  - matching `.dat` pair present: +35
  - `opaque-binary`: +10

- `UnknownBinary`
  - `opaque-binary`: +10
  - fallback evidence: +10 when no stronger known match exists

## Confidence mapping
- `High`: score >= 60
- `Medium`: score >= 25 and < 60
- `Low`: score < 25

## Ordering
Matches are ordered by:
1. Descending numeric score.
2. Stable decoder-id tie-break rank.

This guarantees deterministic output ordering across runs.

## CLI example
```bash
caesar2 --data-dir C:\CAESAR2 --match-signature RESOURCE.DAT
```

Example output shape:

```text
Candidate: RESOURCE.DAT
Class: opaque-binary
Size: 1835008 bytes
Pairing: matching .ix present

Signature matches:
  1. Caesar2DatContainer (score: 60, high)
     - extension hint: .dat extension (+15)
     - pairing hint: matching .ix present (+35)
     - structural hint: opaque-binary (+10)
```
