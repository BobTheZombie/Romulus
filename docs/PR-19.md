# PR 19: Win95 ILBM (.LBM) visible output integration

## Scope
PR 19 keeps the Win95 Caesar II `.LBM` path narrow and debug-oriented:
- parse one ILBM file
- convert decoded indexed pixels + CMAP palette to RGBA
- export one image artifact (PPM)
- view one decoded image in the SDL debug viewer

No renderer framework, resource manager, or simulation logic changes were introduced.

## What was added

### 1) ILBM-to-RGBA conversion helper
`convert_ilbm_to_rgba(const IlbmImageResource&)` now converts parsed ILBM output into `RgbaImage`.

Behavior highlights:
- validates dimensions and indexed buffer length
- fails explicitly if CMAP is empty
- fails explicitly if any decoded pixel index exceeds CMAP bounds
- outputs opaque RGBA (alpha=255) using ILBM CMAP 8-bit channels

### 2) Narrow CLI support for one `.LBM`
Added two single-file debug paths:
- `--export-lbm-file <path> --export-output <path>`
  - decodes one ILBM and exports a PPM via the existing RGBA export helper
- `--view-lbm-file <path>`
  - decodes one ILBM and opens it in the existing SDL debug viewer path

Existing tile export/view options remain unchanged.

### 3) Tests
Extended ILBM tests with conversion coverage:
- successful ILBM -> RGBA conversion
- explicit failure when indexed pixels reference missing CMAP entries

## Usage examples

From a Caesar II data root:

```bash
./caesar2 --export-lbm-file Win95/C2Win95/SomeAsset.LBM --export-output out/some_asset.ppm
```

```bash
./caesar2 --view-lbm-file Win95/C2Win95/SomeAsset.LBM
```

(Use `--smoke-test` with viewer mode in automated/non-interactive checks.)

## Notes
- Bounded parsing/validation from PR 18 is preserved.
- Malformed or unsupported ILBM input continues to fail cleanly with explicit errors.
- This PR intentionally avoids broader asset pipeline abstractions.
